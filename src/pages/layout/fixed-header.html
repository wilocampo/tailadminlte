Fixed Header Layout

This page demonstrates the fixed header layout variant. The top navbar stays pinned at the top of the viewport while the main content scrolls beneath it. This is enabled by adding the fixed-header class to the <body> tag.

Usage: <body class="fixed-header layout-fixed sidebar-expand-lg sidebar-mini sidebar-open">

Try It

Toggle the checkbox and scroll down the page. When enabled, the header stays visible at the top. When disabled, the header scrolls out of view.

How It Works

The fixed-header body class activates CSS rules that make the .app-header element position: sticky with top: 0. Combined with a high z-index, this keeps the header above all content as the user scrolls.

CSS Rules (from app-header.css):
.fixed-header .app-header {
  position: sticky;
  top: 0;
  z-index: var(--z-fixed);
}
