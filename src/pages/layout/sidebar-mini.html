Sidebar Mini Layout

This page demonstrates the sidebar mini mode. When the sidebar is collapsed, it shrinks to show only icons. Hovering over the collapsed sidebar temporarily expands it to reveal the full menu text. This behavior is enabled by the sidebar-mini class on the body (which is present by default in TailAdminLTE).

Usage: <body class="layout-fixed sidebar-expand-lg sidebar-mini sidebar-open">

Try It

First enable "Sidebar Mini mode", then check "Collapse sidebar" to see the mini sidebar. Hover over the collapsed sidebar to see it temporarily expand with a smooth animation.

How It Works

The sidebar-mini class works in conjunction with sidebar-collapse. When both are present on the body:

  • The sidebar shrinks to var(--sidebar-mini-width) (typically around 4.6rem)
  • Menu text, nav arrows, and badges are hidden
  • Only the nav icons remain visible
  • Section headers (.nav-header) are hidden
  • On hover, the sidebar smoothly expands back to full width
  • The brand text also collapses and re-appears on hover

Without sidebar-mini, collapsing the sidebar would slide it completely off-screen instead of showing the icon-only state.

Key CSS Rules (from app-sidebar.css):
/* Collapse to icon-only */
.sidebar-mini.sidebar-collapse .app-sidebar {
  min-width: var(--sidebar-mini-width);
  max-width: var(--sidebar-mini-width);
}

/* Hide text */
.sidebar-mini.sidebar-collapse .sidebar-menu .nav-link p {
  width: 0;
  visibility: hidden;
}

/* Expand on hover */
.sidebar-mini.sidebar-collapse .app-sidebar:hover {
  min-width: var(--sidebar-width);
  max-width: var(--sidebar-width);
}

/* Show text on hover */
.sidebar-mini.sidebar-collapse .app-sidebar:hover .sidebar-menu .nav-link p {
  width: auto;
  visibility: visible;
}
