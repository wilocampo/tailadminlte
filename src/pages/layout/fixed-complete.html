Fixed Complete Layout

This page has all three elements fixed: the sidebar, header, and footer all remain in place while only the main content area scrolls. This provides maximum navigation accessibility.

Usage: <body class="layout-fixed fixed-header fixed-footer sidebar-expand-lg sidebar-mini sidebar-open">

Try It

Toggle each element independently. When all three are enabled, only the main content area scrolls — the sidebar, header, and footer stay fixed in the viewport.

How It Works

The "fixed complete" layout combines three body classes:

  • layout-fixed — Makes the sidebar sticky with its own scrollbar
  • fixed-header — Makes the header/navbar sticky at the top
  • fixed-footer — Makes the footer sticky at the bottom

This is the most "app-like" layout, similar to desktop applications where toolbars and status bars are always visible.

Content Area

Notice how only this content area scrolls. The header stays at the top, the footer at the bottom, and the sidebar on the left — all fixed in place.

Scroll Demo

This layout is ideal for dashboard applications where users need constant access to navigation, actions in the header, and status information in the footer.

More Content

The scrollable content area automatically adjusts its height to fill the space between the fixed header and fixed footer, creating a contained scrolling region.

Additional Block

With all elements fixed, the user never loses access to primary navigation or key actions, regardless of how far they scroll through the content.

Final Block

You've reached the end of the demo content. Try disabling individual fixed elements using the toggles above to see how each one affects the layout.
